# 指令
1. 要件定義.mdを通読して、ゲームの全体像を把握してください。
2. 提供したソースコードを把握して、現在の実装の進み具合を把握してください。
3. 意思決定やシミュレーションに「ブレ」がなく、無機質で機械的なシミュレーションになっています。下記のような要素に「ブレ」を実装し、リアルな世界観を作ってください。
 - NPCの意思決定（発注先や売値など）
 - 顧客の購買ロジック（スコアが高いものしか買われない、ではない）
 - 部品の性能（同じメーカーに頼んでも同じ性能、同じ価格の部品ができるとは限らない）
 - ほかにも揺らぎがあったほうがいいと思われる部分
